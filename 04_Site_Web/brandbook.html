§ 06

Règles d'Usage

Ce que chaque fichier annexe DOIT faire et DOIT éviter pour conserver la cohérence de marque.

✓ À faire
  • Importer les 2 Google Fonts dans chaque <head>
  • Copier le bloc :root complet en entête du CSS
  • Utiliser var(--teal) pour tous les accents actifs
  • Alterner les fonds --bg et --bg-sand entre sections
  • Mettre un mot clé en italique teal dans chaque titre hero
  • Utiliser Cormorant Garamond poids 300 pour tous les titres
  • Appliquer -webkit-font-smoothing: antialiased sur body
  • Inclure progress bar fixe en haut de page
  • Terminer chaque page par footer #132237
  • Utiliser --radius: 4px pour tous les éléments arrondis
  • Respecter --trans: 0.3s ease sur toutes les transitions
  • Ajouter la classe .reveal + IntersectionObserver pour les entrées en scroll
  • Charger odl-styles.css via <link> avant tout <style> local
  • Logo nav : mix-blend-mode: multiply (fond clair) · Logo footer : mix-blend-mode: screen (fond sombre)
  • Protéger le logo avec les classes no-translate et logo-stable
✕ À ne pas faire
  • Coder une couleur en dur si une variable CSS existe déjà
  • Utiliser d'autres polices que Cormorant Garamond + Jost
  • Créer une couleur d'accentuation hors de la palette officielle
  • Utiliser font-weight: bold/700 sur les titres serif
  • Appliquer --peach comme fond de section étendu
  • Utiliser --gold comme couleur principale de section
  • Changer --radius arbitrairement (certains éléments utilisent 6px, 8px, 10px documentés)
  • Mettre le logo en gras ou changer la casse (toujours : "Ocean Drift Lodge")
  • Utiliser un background footer autre que #132237
  • Ajouter des animations non prévues (pas de keyframes arbitraires)
  • Utiliser Inter, Roboto, ou toute autre police système générique
  • Créer des gradients hors du pattern hero documenté
  • Ajouter class="reveal" ou animation: floatLogo sur les logos
  • Redéfinir les variables :root en local si odl-styles.css est chargé

Règles spécifiques par contexte

Logo : Toujours "Ocean Drift" + Lodge en italique teal. Sur fond sombre : "Ocean Drift" blanc + "Lodge" italic teal. Abréviation autorisée : ODL en teal uniquement pour QR / favicon / usage ultra-compact.
Titres H1 : Jamais de majuscules forcées (text-transform:uppercase interdit sur les titres Cormorant). Le poids 300 est obligatoire. La taille minimum est clamp(3rem, 7vw, 6rem).
Section-tag : Toujours en Jost, uppercase, letter-spacing .28em, color --teal. Jamais en --peach ou --navy pour ce rôle.
Bouton CTA principal : Toujours --peach. Le bouton teal est réservé aux confirmations / modales / états actifs. Jamais de bouton --navy en CTA front.
Interdit absolu : Ne jamais utiliser Arial, Helvetica, System-UI, ou toute police fallback comme police affichée. L'import Google Fonts doit toujours précéder le CSS.
Interdit absolu : Ne jamais créer un fond hero en couleur unie (bleu, teal, etc.). Le hero utilise TOUJOURS le gradient tri-couche documenté en §4.

odl-styles.css — CSS commun

Feuille de style partagée chargée par les 15 fichiers HTML du projet. Source unique de vérité pour tous les composants communs : reset, nav, footer, boutons, traduction, logo, animations.

Chargement obligatoire : <link rel="stylesheet" href="odl-styles.css"> dans le <head>, après les fonts Google, avant tout <style> local. Les styles locaux ne couvrent que les composants propres à la page.
Contenu de odl-styles.css : § Variables :root · § Reset · § Progress bar · § Navigation (nav, burger, mobile menu) · § Logo (nav / hero / footer) · § Sections communes · § Boutons · § Footer · § Système de traduction multilingue · § Animations (reveal, pulse, panelIn) · § Utilitaires · § Responsive.
Interdit absolu : Ne jamais redéfinir :root ni les composants communs dans un <style> local si odl-styles.css est chargé. Toute redéfinition crée des conflits de spécificité.

Système de traduction multilingue

Principe : Chaque texte est dupliqué en 5 <span data-lang="xx">. Sans JS, le français est visible par défaut (CSS display:none sur EN/ES/PT/IT). Avec JS, la classe js-lang est posée sur <html> et setLang() contrôle la visibilité via .lang-visible.
Protection logo : Les classes no-translate et logo-stable exemptent le logo du système de masquage. Ne jamais oublier ces classes sur les <img> et <picture> logo.
